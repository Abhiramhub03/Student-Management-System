1. Project Overview & Architecture

This project implements a Student–Course Management System using Spring Boot 3, Spring MVC, Spring Data JPA (Hibernate), JSP/JSTL for views, and MySQL as the relational database. The application supports full Create, Read, and Update operations for two related entities: Student and Course, along with an inner-join query, exception handling, and comprehensive unit tests.

1.1 Application Architecture

The application follows the standard Layered (N-Tier) Architecture used in enterprise Spring MVC applications:

🌐 View Layer (JSP + JSTL + CSS)
JSP pages handle all UI rendering. JSTL tags and EL expressions are used for dynamic data binding. CSS provides responsive, visually appealing styling.
🎮 Controller Layer (Spring MVC)
Handles HTTP requests, binds form data to model objects, calls the service layer, and forwards results to the appropriate JSP view.
⚙️ Service Layer
Contains business logic. Acts as a bridge between controller and repository. Handles transaction management and validation rules.
🗄️ Repository Layer (Spring Data JPA)
Extends JpaRepository for CRUD operations. Contains custom JPQL queries including the required inner-join between Student and Course.

4. Repository Layer

Both repositories extend JpaRepository, providing built-in CRUD operations. A custom JPQL query performs an inner join between Student and Course.

CourseRepository.java

package com.example.studentcourse.repository;

@Repository
public interface CourseRepository extends JpaRepository<Course, Long> {

    @Query("SELECT c FROM Course c WHERE c.courseCode = :code")
    Optional<Course> findByCourseCode(@Param("code") String code);

    List<Course> findByCourseName(String courseName);
}

StudentRepository.java (with Inner Join Query)

package com.example.studentcourse.repository;

@Repository
public interface StudentRepository extends JpaRepository<Student, Long> {

    // Custom JPQL inner join query – returns student + course info
    @Query("SELECT s FROM Student s INNER JOIN s.course c WHERE c.id = :courseId")
    List<Student> findStudentsByCourseId(@Param("courseId") Long courseId);

    // Inner join projection – returns student name + course name as Object[]
    @Query("SELECT s.firstName, s.lastName, c.courseName, c.instructor "
         + "FROM Student s INNER JOIN s.course c")
    List<Object[]> findAllStudentsWithCourseDetails();

    Optional<Student> findByEmail(String email);

    List<Student> findByCourseId(Long courseId);
}
Inner Join Query: The method findAllStudentsWithCourseDetails() performs a JPQL inner join that matches every student to their enrolled course. This returns a list of Object[] arrays, each holding the student's name, the course name, and the instructor — which is then consumed by the service layer and displayed in the read view.

5. Service Layer

StudentService.java

package com.example.studentcourse.service;

@Service
@Transactional
public class StudentService {

    @Autowired private StudentRepository studentRepository;
    @Autowired private CourseRepository  courseRepository;

    public List<Student> getAllStudents() {
        return studentRepository.findAll();
    }

    public Student getStudentById(Long id) {
        return studentRepository.findById(id)
            .orElseThrow(() -> new EntityNotFoundException("Student not found: " + id));
    }

    public Student saveStudent(Student student) {
        if (studentRepository.findByEmail(student.getEmail()).isPresent()) {
            throw new DuplicateEmailException("Email already registered: " + student.getEmail());
        }
        return studentRepository.save(student);
    }

    public Student updateStudent(Long id, Student updated) {
        Student existing = getStudentById(id);
        existing.setFirstName(updated.getFirstName());
        existing.setLastName(updated.getLastName());
        existing.setEmail(updated.getEmail());
        existing.setPhone(updated.getPhone());
        existing.setEnrollmentDate(updated.getEnrollmentDate());
        existing.setCourse(updated.getCourse());
        return studentRepository.save(existing);
    }

    public List<Object[]> getStudentsWithCourseDetails() {
        return studentRepository.findAllStudentsWithCourseDetails();
    }
}

CourseService.java

package com.example.studentcourse.service;

@Service
@Transactional
public class CourseService {

    @Autowired private CourseRepository courseRepository;

    public List<Course> getAllCourses() {
        return courseRepository.findAll();
    }

    public Course getCourseById(Long id) {
        return courseRepository.findById(id)
            .orElseThrow(() -> new EntityNotFoundException("Course not found: " + id));
    }

    public Course saveCourse(Course course) {
        if (courseRepository.findByCourseCode(course.getCourseCode()).isPresent()) {
            throw new DuplicateCourseCodeException("Course code already exists: " + course.getCourseCode());
        }
        return courseRepository.save(course);
    }

    public Course updateCourse(Long id, Course updated) {
        Course existing = getCourseById(id);
        existing.setCourseName(updated.getCourseName());
        existing.setCourseCode(updated.getCourseCode());
        existing.setInstructor(updated.getInstructor());
        existing.setCredits(updated.getCredits());
        existing.setDescription(updated.getDescription());
        return courseRepository.save(existing);
    }
}

6. Controller Layer

6.1 Create Operation – StudentController

package com.example.studentcourse.controller;

@Controller
@RequestMapping("/students")
public class StudentController {

    @Autowired private StudentService studentService;
    @Autowired private CourseService  courseService;

    // ── Show add-student form ──
    @GetMapping("/add")
    public String showAddForm(Model model) {
        model.addAttribute("student", new Student());
        model.addAttribute("courses", courseService.getAllCourses());
        return "student/add-student";
    }

    // ── Handle form submission (Create) ──
    @PostMapping("/add")
    public String addStudent(@ModelAttribute("student") @Valid Student student,
                              BindingResult bindingResult,
                              @RequestParam("courseId") Long courseId,
                              Model model,
                              RedirectAttributes redirectAttrs) {
        if (bindingResult.hasErrors()) {
            model.addAttribute("courses", courseService.getAllCourses());
            return "student/add-student";
        }
        try {
            Course course = courseService.getCourseById(courseId);
            student.setCourse(course);
            studentService.saveStudent(student);
            redirectAttrs.addFlashAttribute("successMsg", "Student added successfully!");
            return "redirect:/students/list";
        } catch (DuplicateEmailException e) {
            model.addAttribute("errorMsg", e.getMessage());
            model.addAttribute("courses", courseService.getAllCourses());
            return "student/add-student";
        }
    }
}

8. Exception Handling

Custom exceptions are defined and a global @ControllerAdvice handles them gracefully, showing user-friendly error pages instead of stack traces.

// Custom exception for duplicate email
public class DuplicateEmailException extends RuntimeException {
    public DuplicateEmailException(String msg) { super(msg); }
}

// Custom exception for duplicate course code
public class DuplicateCourseCodeException extends RuntimeException {
    public DuplicateCourseCodeException(String msg) { super(msg); }
}

// Global exception handler
@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(EntityNotFoundException.class)
    public ModelAndView handleEntityNotFound(EntityNotFoundException ex) {
        ModelAndView mav = new ModelAndView("error/not-found");
        mav.addObject("message", ex.getMessage());
        return mav;
    }

    @ExceptionHandler(DataIntegrityViolationException.class)
    public ModelAndView handleDataIntegrity(DataIntegrityViolationException ex) {
        ModelAndView mav = new ModelAndView("error/integrity-error");
        mav.addObject("message",
            "Data integrity violation. This record may already exist or a required field is missing.");
        return mav;
    }

    @ExceptionHandler(Exception.class)
    public ModelAndView handleGeneral(Exception ex) {
        ModelAndView mav = new ModelAndView("error/general-error");
        mav.addObject("message", ex.getMessage());
        return mav;
    }
}
Integrity Violations handled: Duplicate email (UNIQUE constraint on student.email), duplicate course code (UNIQUE constraint on course.course_code), and null violations on NOT NULL columns. All are caught and shown as friendly messages in the UI rather than HTTP 500 errors.

11. Challenges & Solutions

Challenge 1: JSP Not Rendering in Spring Boot

Spring Boot's default embedded Tomcat does not support JSP rendering without additional configuration. Initial setup returned blank pages or 404 errors when loading JSP views.

Added tomcat-embed-jasper dependency in pom.xml and configured spring.mvc.view.prefix and spring.mvc.view.suffix in application.properties. Also ensured JSPs were placed under src/main/webapp/WEB-INF/views/ (not under resources/).

Challenge 2: LazyInitializationException on Student.course

When fetching students in the list view, accessing student.course.courseName via EL caused a LazyInitializationException because the session had already closed.

Changed the @ManyToOne fetch type from LAZY to EAGER on the Student entity. Alternatively, added @Transactional to the service method and used JOIN FETCH in the repository query for better performance.

Challenge 3: Form Binding for Nested Course Object

Binding the selected course (from a dropdown) to the Student model object was problematic because the form sent a courseId (Long) but the Student entity required a full Course object.

Added a separate @RequestParam("courseId") parameter in the controller, then fetched the full Course entity via service and set it on the student object before saving.

Challenge 4: Handling DataIntegrityViolationException

When a duplicate email was submitted, Hibernate threw a DataIntegrityViolationException at the database level, bypassing service-layer validation in some cases.

Added a pre-save check in the service layer using findByEmail() to throw a custom DuplicateEmailException before the save. The GlobalExceptionHandler also catches the Hibernate-level exception as a fallback.

Challenge 5: Inner Join Query Returning Object[]

The JPQL inner join projection returned List<Object[]> which needed to be iterated correctly in the JSP using EL. Standard EL with ${row.property} doesn't work for array elements.

Used indexed EL notation ${row[0]}, ${row[1]}, etc., in the JSP's <c:forEach> loop. Alternatively, a DTO/projection interface could be used for type-safe access.
